# 🏠 不動産チャットボット — 夜間反響・内見機会損失防止システム

---

## 【対象業種】

従業員5〜30名規模の不動産仲介会社（賃貸メイン）向けチャットボット

## 【解決する課題】

不動産業界では、ポータルサイト経由の反響の**約40%が営業時間外**（18時以降・休日）に発生します。しかし、多くの仲介会社では翌営業日まで対応できず、その間にユーザーは競合他社へ流れてしまいます。

具体的には以下の機会損失が発生しています。

- **夜間・休日の物件問い合わせに即応できない**ことによる離脱 → 推定**月20〜30件の反響ロス**（1件あたりの仲介手数料を考慮すると**月30〜50万円相当の機会損失**）
- **「初期費用は？」「ペット可？」「審査基準は？」といった定型質問**への対応に、スタッフが1日あたり約1.5〜2時間を消費 → 追客・内見対応など**売上に直結する業務の時間が圧迫**される
- **内見予約の電話対応ができない**時間帯に、見込み客の検討熱が冷めてしまう → 来店率・成約率の低下

## 【主な機能】

### ① 物件検索の即時回答（24時間365日）

営業時間外でも、ユーザーが「渋谷区でペット可の2LDK」と入力するだけで、条件に合った物件を**最大3件まで比較形式で即座に提示**します。地域・予算・間取り・設備（ペット可・オートロック・駐車場等）など、複数条件の掛け合わせ検索に対応しています。

### ② FAQ自動応答

「初期費用はいくら？」「保証人は必要？」「契約期間は？」など、**頻出する定型質問20件以上に即座に回答**します。スタッフが繰り返し答えていた質問をチャットボットが肩代わりすることで、対応工数を大幅に削減します。ナレッジベースの更新だけで回答内容を追加・修正できるため、運用負荷も最小限です。

### ③ 内見予約の自動受付 → Google Sheets記録 ＋ メール通知

ユーザーが「内見したい」と入力すると、物件名・氏名・電話番号・メールアドレス・希望日時を会話形式で収集します。**すべての情報を一括入力した場合は即座に予約確認を返答**し、不足情報がある場合は段階的にヒアリングします。

予約情報はGoogle Sheetsに自動記録され、同時に担当者へメール通知が送信されるため、**翌朝出社した時点で予約一覧が揃っている**状態を実現します。

## 【使用した技術】

| 技術要素 | 詳細 |
|---------|------|
| **プラットフォーム** | Dify（Advanced Chatモード） |
| **LLM** | GPT-4o / GPT-4o-mini（用途に応じて使い分け） |
| **意図分類** | LLMノードによる3分岐ルーティング（物件検索 / FAQ / 内見予約） |
| **RAG検索** | 物件データ：ハイブリッド検索（セマンティック70% + キーワード30%）、FAQデータ：セマンティック検索 |
| **埋め込みモデル** | text-embedding-3-large |
| **会話状態管理** | Conversation Variable（予約フロー中の文脈保持） |
| **条件分岐** | IF/ELSEノードによるインテント＋状態ベースの分岐制御 |
| **外部連携** | Google Apps Script → Google Sheets記録 ＋ Gmail通知 |
| **埋め込み** | JavaScript Widgetによる既存Webサイトへの設置 |

### 設計上の工夫

- **意図分類の精度向上**：LLMにカテゴリごとの具体例を明示し、temperature 0.3の低温設定で分類のブレを抑制
- **検索精度の最適化**：物件検索では多様な表現に対応するためハイブリッド検索を採用し、FAQでは意味的な一致を重視するセマンティック検索を採用。それぞれスコア閾値を設定し、関連性の低い結果を除外
- **予約フローの文脈維持**：Conversation Variableで予約ステータスを管理し、情報収集の途中でもユーザーの入力を正しく予約フローに振り分ける
- **一括入力・段階入力の両対応**：「物件名＋名前＋電話＋メール＋日時」を一度に入力した場合は即座に確認を返し、部分的な入力には不足情報のみを質問するUX設計

## 【想定される効果】

| 指標 | 現状（推定） | 導入後（推定） | 改善幅 |
|------|------------|--------------|--------|
| 夜間・休日の反響対応 | 翌営業日まで未対応 | 即時自動対応 | **反響ロス月20〜30件を削減** |
| 定型質問の対応時間 | 月40〜60時間 | 月10〜15時間 | **約70%削減** |
| 内見予約の受付漏れ | 電話不通時は機会損失 | 24時間自動受付 | **夜間予約の取りこぼしゼロ** |
| 機会損失額 | 月30〜50万円相当 | 大幅削減 | **月額コストに対して十分な投資対効果** |

## 【テスト結果】

開発段階で3フェーズにわたるテストを実施し、最終フェーズで**合格率92.3%（13件中12件合格）**を達成しました。物件検索・FAQ・内見予約の各機能について、正常系・異常系の両方を検証しています。
