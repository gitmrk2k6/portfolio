# 📂 portfolio (ポートフォリオWebサイト)

本ディレクトリは、Keisuke（AIエージェント開発 × 業務効率化）のポートフォリオサイトのソースコードを管理するフォルダです。

---

## 🎯 このプロジェクトの「目的」
* **自身のスキルと価値観の証明**
  元教員からインフラエンジニア、そしてAIチャットボット開発者へと転身した経歴と、「AIで人々の本業に時間を取り戻す」という信念を伝えること。
* **実績の可視化と認知拡大**
  これまで作成してきたAIチャットボットはもちろん、今後追加される多様なプロジェクト（業務改善ツール、Webアプリなど）を含め、あらゆる実績を並列に美しく展示する受け皿とすること。
* **新たな案件・キャリアへの接続**
  サイトを訪れたクライアントや採用担当者に信頼感を与え、相談や依頼（お問い合わせ）へスムーズに誘導すること。

## 📦 このプロジェクトの「成果物」
* **モダンで魅力的なWebアプリケーション**
  React + Vite を用いて構築された、高速で美しいポートフォリオサイト。（UI/UXが考慮されたシングルページ、または複数ページのサイト）
* **レスポンシブ対応済みのUI**
  PC、タブレット、スマートフォンなど、どのデバイスから見ても崩れず、快適に閲覧できるデザイン。
* **保守性・拡張性の高いソースコード**
  将来的に新しい実績やコンテンツ（チャットボット以外のサービス、ブログ記事など）を追加しても、レイアウトが崩れず容易に拡張できる柔軟なコンポーネント設計。

## ✅ 「完了した」と言える条件（Definition of Done）
1. `portfolio_draft.md` に記載されているすべての情報（自己紹介、実績、対応業務、ビジョンなど）がWebサイト上に漏れなく実装・デザインされていること。
2. 各デバイス（モバイル・PC）での表示崩れがなく、意図した通りにアニメーションや遷移が動作すること。
3. 問い合わせやSNSなどの「次のアクション（Call to Action）」への導線が機能していること。
4. 本番環境（Vercel, Netlify, GitHub Pagesなど）へ正常にデプロイされ、インターネット上で誰でも閲覧できる状態になっていること。

---

## 🛠 フォルダ構成と役割
* `README.md` : このファイル。プロジェクトの目的と全体像（人間向け）。
* `instruction.md` : AIアシスタントへの指示書。開発をサポートするAIに読ませるルール。
* `portfolio_draft.md` : サイトに掲載するテキストコンテンツの大元となる原稿。
* `project_descriptions/` : 各プロジェクトの詳細な説明文を格納するディレクトリ（将来の実績追加にも対応）。
* `src/` : Reactコンポーネントやスタイルシート、ロジックが含まれるメインのソースコードディレクトリ。
* `public/` : 画像などの静的アセット。
